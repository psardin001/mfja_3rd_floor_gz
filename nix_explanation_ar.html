شرح ملفات Nix في مشروع MFJA

يوجد ملفان في جذر المشروع: flake.nix و flake.lock. الأول هو الملف الذي نعدله عادة، والثاني هو ملف قفل تلقائي يثبت النسخ الدقيقة حتى كل شخص يدخل نفس بيئة التطوير.

هذا المشروع يستخدم Nix بشكل hybrid: Nix يوفر أدوات البناء وبعض المكتبات، لكن ROS 2 Jazzy وGazebo Harmonic يأتون من تثبيت النظام على الجهاز، وليس من Nix.

الفكرة السريعة كيف أستخدمها؟ شرح flake.nix شرح flake.lock ماذا يحدث عند nix develop؟ متى أعدل كل ملف؟ أخطاء شائعة ملخص سريع

1. الفكرة السريعة

الملف وظيفته هل أعدله يدوياً؟
flake.nix يصف بيئة التطوير: من أين نأخذ packages، ما هي الأدوات المطلوبة، ما هي متغيرات البيئة، وماذا يحدث عند دخول shell. نعم، عندما تريد إضافة أداة أو تعديل shell.
flake.lock يثبت النسخ الدقيقة للـ inputs، حالياً nixpkgs. هذا يجعل البيئة قابلة للتكرار عند كل شخص. لا تعدله باليد. Nix يحدثه بأوامر update/lock.

تخيل flake.nix كأنه recipe. أما flake.lock فهو receipt فيه رقم commit والـ hash الدقيقين للمواد التي استُخدمت.

2. كيف أستخدمها؟

من جذر الريبو الذي يحتوي flake.nix: 

cd /home/tiago/mfja_3rd_floor_ros2_ws/src/mfja_3rd_floor_gz
nix develop

بعد الدخول، الـ shell يعمل setup للبيئة ويطبع رسالة. للبناء، الملف نفسه يقترح البناء من جذر workspace: 

cd ../..
colcon build --symlink-install --base-paths src/mfja_3rd_floor_gz

هذه البيئة لا تثبت ROS 2 تلقائياً. يجب أن يكون /opt/ros/jazzy/setup.bash موجوداً على الجهاز. إذا غير موجود، ستدخل shell لكن سيظهر تحذير أن ROS 2 Jazzy غير مثبت.

3. شرح flake.nix

الوصف

description = "Hybrid development shell for the MFJA 3rd floor ROS 2 Jazzy / Gazebo Harmonic simulation";

نص تعريفي فقط. لا يغير السلوك. يوضح أن الهدف shell تطوير هجينة لـ ROS 2 Jazzy وGazebo Harmonic.

inputs

inputs = {
  nixpkgs.url = "github:NixOS/nixpkgs/nixos-25.11";
};

هذا يقول لـ Nix: استخدم nixpkgs من GitHub branch nixos-25.11. لكن النسخة الدقيقة لا تؤخذ من هنا وحده، بل من flake.lock.

outputs

outputs = { nixpkgs, ... }:
  let
    ...
  in
  {
    devShells = ...
  };

هذا هو الجزء الذي ينتج الأشياء التي يمكن لـ Nix استخدامها. في مشروعنا الناتج الأساسي هو devShells، أي بيئة تطوير تدخلها بـ nix develop.

supportedSystems

supportedSystems = [
  "x86_64-linux"
  "aarch64-linux"
];

يحدد المعماريات المدعومة:

  • x86_64-linux: أجهزة Linux العادية 64-bit.
  • aarch64-linux: ARM 64-bit، مثل بعض الأجهزة أو boards.

forAllSystems

forAllSystems = nixpkgs.lib.genAttrs supportedSystems;

helper من Nix ينشئ نفس dev shell لكل نظام موجود في supportedSystems. بدل تكرار الكود مرتين، نكتبه مرة واحدة.

pkgs

pkgs = import nixpkgs {
  inherit system;
};

هنا يتم تحميل package set الخاص بالنظام الحالي. يعني إذا جهازك x86_64-linux، سيجلب packages المناسبة لهذه المعمارية.

pythonEnv

pythonEnv = pkgs.python312.withPackages (ps: with ps; [
  catkin-pkg
  empy
  lark
  numpy
  pyyaml
  setuptools
]);

ينشئ Python 3.12 مع packages يحتاجها build أو scripts:

الحزمة وظيفتها العامة
catkin-pkgأدوات Python لفهم metadata الخاصة بحزم ROS/catkin.
empyقوالب code generation مستخدمة كثيراً في ROS.
larkParser library، قد تحتاجها أدوات build أو scripts.
numpyعمليات عددية ومصفوفات.
pyyamlقراءة وكتابة YAML.
setuptoolsتغليف وتثبيت packages Python.

spdlogRos

spdlogRos = (pkgs.spdlog.override {
  fmt = pkgs.fmt_9;
}).overrideAttrs (...)

هذا يجهز نسخة spdlog مناسبة لبيئة ROS. المهم هنا:

  • يستخدم fmt_9 بدل fmt الافتراضي.
  • يثبت نسخة spdlog على 1.12.0.
  • يجلب المصدر من GitHub من tag v1.12.0.
  • يعطل الاختبارات بـ doCheck = false لتخفيف build داخل shell.

tinyxml2Ros

tinyxml2Ros = pkgs.tinyxml-2.overrideAttrs (...)

يثبت tinyxml2 على نسخة 10.0.0. ROS وبعض أدواته قد تحتاج ABI أو نسخة متوقعة من هذه المكتبة.

rosRuntimeLibs

rosRuntimeLibs = pkgs.lib.makeLibraryPath [
  pkgs.fmt_9
  pkgs.lttng-ust.out
  pkgs.openssl
  spdlogRos
  pkgs.stdenv.cc.cc.lib
  tinyxml2Ros
];

هذا يصنع string لمسارات المكتبات. لاحقاً يوضع في LD_LIBRARY_PATH حتى البرامج التي تعمل داخل shell تجد هذه المكتبات وقت التشغيل.

colconWrapper

colconWrapper = pkgs.writeShellScriptBin "colcon" ''
  if [ -x /usr/bin/colcon ]; then
    exec /usr/bin/colcon "$@"
  fi

  echo "colcon is not installed on the host." >&2
  ...
'';

هذا لا يثبت colcon من Nix. هو wrapper باسم colcon:

  • إذا وجد /usr/bin/colcon، يستخدمه مباشرة.
  • إذا لم يجده، يطبع رسالة تقول ثبته بـ apt.

السبب: البيئة هجينة وتريد استخدام ROS/colcon من النظام.

mkShell packages

packages = [
  pkgs.bashInteractive
  pkgs.cmake
  colconWrapper
  pkgs.gcc
  pkgs.fmt_9
  pkgs.git
  pkgs.gnumake
  pkgs.lttng-ust.out
  pkgs.ninja
  pkgs.openssl
  pkgs.pkg-config
  spdlogRos
  pkgs.stdenv.cc.cc.lib
  tinyxml2Ros
  pythonEnv
];

هذه هي الأدوات والمكتبات التي تظهر داخل shell. أهمها:

العنصر لماذا موجود؟
cmake, gnumake, ninjaأدوات build.
gccمترجم C/C++.
gitللتعامل مع repo وdependencies.
pkg-configليجد build system مسارات المكتبات.
fmt_9, spdlogRos, tinyxml2Rosمكتبات C++ يحتاجها ROS/build runtime.
pythonEnvPython مع packages المطلوبة.

shellHook

shellHook = ''
  export ROS_DISTRO=jazzy
  export MFJA_NIX_MODE=hybrid
  export RMW_IMPLEMENTATION="${RMW_IMPLEMENTATION:-rmw_fastrtps_cpp}"
  export LD_LIBRARY_PATH="${rosRuntimeLibs}${LD_LIBRARY_PATH:+:}${LD_LIBRARY_PATH:-}"

  if [ -f /opt/ros/jazzy/setup.bash ]; then
    source /opt/ros/jazzy/setup.bash
    ...
  fi
'';

هذا الكود يعمل تلقائياً عند دخول nix develop.

  • ROS_DISTRO=jazzy: يحدد ROS distro.
  • MFJA_NIX_MODE=hybrid: علامة داخلية تفيد أنك داخل shell هجينة.
  • RMW_IMPLEMENTATION: إذا غير محدد، يستخدم rmw_fastrtps_cpp.
  • LD_LIBRARY_PATH: يضيف مكتبات Nix runtime.
  • source /opt/ros/jazzy/setup.bash: يدخل ROS 2 Jazzy من النظام.

4. شرح flake.lock

هذا ملف JSON يولده Nix. وظيفته تثبيت inputs حتى لا تتغير كل مرة تشغل nix develop. 

{
  "nodes": {
    "nixpkgs": {
      "locked": {
        "lastModified": 1779102034,
        "narHash": "sha256-vZJ...",
        "owner": "NixOS",
        "repo": "nixpkgs",
        "rev": "687f05a9184cad4eaf905c48b63649e3a86f5433",
        "type": "github"
      },
      "original": {
        "owner": "NixOS",
        "ref": "nixos-25.11",
        "repo": "nixpkgs",
        "type": "github"
      }
    }
  }
}
الحقل معناه
nodes كل dependency في flake. حالياً عندنا nixpkgs و root.
locked.rev رقم commit الدقيق من nixpkgs الذي سيستخدمه المشروع.
locked.narHash hash يستخدمه Nix للتأكد أن المحتوى الذي نزل هو نفسه المتوقع.
locked.lastModified وقت آخر تعديل للـ input بصيغة Unix timestamp.
original.ref الفرع أو المرجع الذي طلبناه في flake.nix، هنا nixos-25.11.
root.inputs يقول إن flake الرئيسي يعتمد على node اسمها nixpkgs.
version نسخة schema الخاصة بملف lock. ليست نسخة المشروع.

5. ماذا يحدث عند nix develop؟

1. قراءة flake.nix Nix يعرف أن المطلوب devShells.default.
2. قراءة flake.lock يثبت nixpkgs على commit محدد.
3. بناء shell يجلب cmake, gcc, pythonEnv, spdlog, tinyxml2...
4. shellHook يضبط ROS_DISTRO وLD_LIBRARY_PATH ويعمل source لـ ROS.
5. أنت تبني تشغل colcon من workspace root.

النتيجة: كل شخص عنده Nix وROS 2 Jazzy مثبتين على النظام يحصل على نفس أدوات build تقريباً ونفس nixpkgs commit.

6. متى أعدل كل ملف؟

عدل flake.nix إذا:

  • تريد إضافة أداة مثل clang-format أو gdb.
  • تريد إضافة Python package إلى pythonEnv.
  • تريد تعديل متغير بيئة داخل shellHook.
  • تريد تغيير نسخة مكتبة مثل spdlog أو tinyxml2.

لا تعدل flake.lock يدوياً

إذا أردت تحديث nixpkgs، استخدم: 

nix flake update

أو لتحديث input محدد: 

nix flake lock --update-input nixpkgs

تحديث flake.lock قد يغير نسخ packages، لذلك بعده شغل build وتأكد أن ROS workspace ما زال يبني.

7. أخطاء شائعة

المشكلة السبب المحتمل الحل
nix: command not found Nix غير مثبت على الجهاز. ثبت Nix أولاً.
رسالة عن experimental features flakes غير مفعلة في إعدادات Nix. nix --extra-experimental-features "nix-command flakes" develop
تحذير أن /opt/ros/jazzy/setup.bash غير موجود ROS 2 Jazzy غير مثبت من apt. ثبت ROS 2 Jazzy وROS-Gazebo packages على النظام.
colcon is not installed on the host الـ wrapper لم يجد /usr/bin/colcon. sudo apt install python3-colcon-common-extensions
أخطاء shared libraries وقت التشغيل مكتبة غير موجودة في LD_LIBRARY_PATH أو mismatch بين host وNix. ادخل من nix develop وتأكد أن LD_LIBRARY_PATH محدث.

8. ملخص سريع

  • flake.nix هو الملف الذي يصف بيئة التطوير.
  • flake.lock يثبت نسخة nixpkgs الدقيقة.
  • هذه البيئة هجينة: Nix للأدوات، ROS/Gazebo من apt على الجهاز.
  • nix develop يدخل shell ويعمل source لـ /opt/ros/jazzy/setup.bash إذا موجود.
  • colcon داخل shell هو wrapper يستخدم /usr/bin/colcon.
  • أضف الأدوات إلى packages، وأضف Python packages إلى pythonEnv.
  • لا تعدل flake.lock باليد. حدثه بأوامر Nix.
