This page keeps only the important practical scenarios: running Room 315, controlling shuttles, testing switches and stoppers, reading sensor feedback, running the full third floor, and testing robot motion and runtime checks.

Terminal Setup

Build once after changing code, messages, config, or launch files. The commands use ${MFJA_WS:-$HOME/test_mfja_ws}, so set MFJA_WS only if your workspace is somewhere else.

Build

Build workspace
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash

colcon build --symlink-install

source install/setup.bash

Source Only

New terminal
cd "${MFJA_WS:-$HOME/test_mfja_ws}"

Scenario 1: Room 315 with Both Rails and One Shuttle Per Side

This is the main training scenario: Room 315 only, both right and left rails enabled, one visible shuttle on each rail, and both shuttles waiting until you send ON.

Terminal 1: launch Room 315
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 launch mfja_room_315_bringup room_315_only.launch.py \
  robots:=none \
  start_paused:=false \
  gui:=true \
  enable_room315_kinematic_shuttles:=true \
  enable_room315_right_rail:=true \
  enable_room315_left_rail:=true \
  room315_right_shuttle_count:=1 \
  room315_left_shuttle_count:=1 \
  room315_right_start_slot:=2 \
  room315_left_start_slot:=2 \
  room315_shuttles_start_enabled:=false \
  room315_shuttle_speed:=0.2 \
  room315_show_device_markers:=true
Terminal 2: start / stop left shuttle
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 topic pub --once /room_315/rails/left/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_left_shuttle_1', command: 'ON'}"

ros2 topic pub --once /room_315/rails/left/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_left_shuttle_1', command: 'OFF'}"
Terminal 2: start / stop right shuttle
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_right_shuttle_1', command: 'ON'}"

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_right_shuttle_1', command: 'OFF'}"
Terminal 3: watch shuttle states
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 topic echo /room_315/rails/right/shuttles/state \
  mfja_rail_interfaces/msg/ShuttleState

# In another terminal if needed:
ros2 topic echo /room_315/rails/left/shuttles/state \
  mfja_rail_interfaces/msg/ShuttleState

Scenario 2: Add, Remove, Reset, and Control Shuttles

Use this scenario to test the full shuttle lifecycle. Adding a shuttle is a service call: start_enabled: false spawns it stopped, and start_enabled: true spawns it moving immediately.

Commands for one shuttle
Add / ON / OFF / RESET / REMOVE
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 service call /room_315/rails/right/shuttles/add \
  mfja_rail_interfaces/srv/AddShuttle \
  "{name: 'room315_right_test_1', start_slot: '3', speed: 0.2, start_enabled: false}"

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_right_test_1', command: 'ON'}"

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_right_test_1', command: 'OFF'}"

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_right_test_1', command: 'RESET'}"

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'room315_right_test_1', command: 'REMOVE'}"
Commands for all shuttles on a rail
Control ALL shuttles
ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'ALL', command: 'OFF'}"

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'ALL', command: 'ON'}"

ros2 topic pub --once /room_315/rails/right/shuttles/command \
  mfja_rail_interfaces/msg/ShuttleCommand \
  "{name: 'ALL', command: 'RESET'}"

Scenario 3: Change Switch Positions and Test Stoppers

Switch states are EXTERIOR and INTERIOR. Stoppers use 1 to close and stop a shuttle, and 0 to open and allow passage.

Switch route test
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 topic pub --once /room_315/rails/right/switches/command \
  mfja_rail_interfaces/msg/SwitchCommand \
  "{switches: [{name: 'ALL', state: 'EXTERIOR'}]}"

ros2 topic pub --once /room_315/rails/right/switches/command \
  mfja_rail_interfaces/msg/SwitchCommand \
  "{switches: [{name: 'A1', state: 'INTERIOR'}]}"

ros2 topic echo /room_315/rails/right/switches/state \
  mfja_rail_interfaces/msg/SwitchState
Stopper close / open test
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 topic pub --once /room_315/rails/right/stoppers/command \
  mfja_rail_interfaces/msg/StopperCommand \
  "{stoppers: [{name: 'A1', state: '1'}]}"

ros2 topic echo /room_315/rails/right/stoppers/state \
  mfja_rail_interfaces/msg/StopperState

ros2 topic pub --once /room_315/rails/right/stoppers/command \
  mfja_rail_interfaces/msg/StopperCommand \
  "{stoppers: [{name: 'A1', state: '0'}]}"
Practical flow: run a slow shuttle, close A1, wait until it stops, change the switch route, then open A1 so the shuttle continues.

Quick Checks When Something Fails

Inspect topics and messages

ROS API check
cd "${MFJA_WS:-$HOME/test_mfja_ws}"
source /opt/ros/jazzy/setup.bash
source install/setup.bash

ros2 topic list | grep /room_315/rails
ros2 service list | grep /room_315/rails
ros2 topic list | grep -E '^/(kuka1|staubli1|yaskawa_hc10_1|yaskawa_hc10dt_1|tiago1)/'
ros2 interface show mfja_rail_interfaces/srv/AddShuttle
ros2 interface show mfja_rail_interfaces/msg/ShuttleCommand
ros2 interface show mfja_rail_interfaces/msg/SwitchCommand
ros2 interface show mfja_rail_interfaces/msg/StopperCommand
ros2 interface show mfja_rail_interfaces/msg/SensorFeedback
ros2 interface show trajectory_msgs/msg/JointTrajectory
ros2 interface show geometry_msgs/msg/Twist
If a new shuttle does not appear, the slot may already be occupied or the name may be duplicated. If a switch moves visually but the shuttle does not change route, watch /switches/state and confirm the command went to the correct rail: right or left.
